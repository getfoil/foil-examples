# Customer Service Agent

A sample customer service agent for "TechStore" that demonstrates [Foil](https://getfoil.com) observability with OpenAI tool calling.

The agent can look up orders, check product availability, initiate refunds, and create support tickets. It runs as either a CLI chatbot or a web app with a chat UI.

## Features

- **Tool calling** — the agent uses OpenAI function calling with an agentic tool loop
- **Multimodal input** — the CLI supports image attachments (e.g. damaged product photos)
- **Web UI** — Express server with a browser-based chat interface
- **Foil tracing** — every conversation is captured as a single trace with LLM and tool spans

## Setup

```bash
npm install
```

Create a `.env` file:

```
OPENAI_API_KEY=sk-...
FOIL_API_KEY=sk_live_...
```

## Usage

### CLI

```bash
node index.js
```

Type messages to chat. Attach images with `/image <path> <message>`. Type `/quit` to exit.

### Web UI

```bash
node web.js
```

Open [http://localhost:3005](http://localhost:3005) in your browser.

## How Foil Tracing Works

This example uses **manual tracing** with `ctx.llmCall()` and `ctx.executeTools()` — no `instrumentModules`.

### CLI (`index.js`)

The entire interactive loop runs inside a single `foil.trace()` call. Each user turn creates an LLM span via `ctx.llmCall()`, and tool executions are traced with `ctx.executeTools()`.

### Web (`web.js`)

Uses the **deferred-promise pattern** to keep one Foil trace alive per browser session:

1. On the first request, `foil.trace()` starts a trace and returns a promise that stays pending
2. The trace context (`ctx`) is stored in a `sessionTraces` Map keyed by session ID
3. Subsequent requests reuse the same `ctx` for `llmCall()` and `executeTools()`
4. When the session ends (browser close, timeout, or new session), the deferred promise resolves and the trace completes

This ensures all turns in a multi-turn conversation appear under a single trace.

## Available Tools

| Tool | Description |
|------|-------------|
| `lookup_order` | Look up order status, tracking, and details by order ID |
| `check_product_availability` | Check if a product is in stock with price and warranty info |
| `initiate_refund` | Initiate a refund for a delivered order |
| `create_support_ticket` | Create a support ticket for issues needing human follow-up |
